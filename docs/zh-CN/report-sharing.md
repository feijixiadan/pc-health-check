# 报告分享与隐私检查

PC Health Check 默认离线、只读、不上传数据。但报告仍可能包含本机信息。只要报告会离开电脑主人的私密支持对话，就应该先使用去身份化模式，并人工检查一遍。

## 推荐命令

在仓库目录运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\pc-health-check.ps1 -RedactIdentity -OpenReport
```

这个模式会尽量隐藏本地身份信息，同时保留诊断所需的数量、状态和健康信号。

## `-RedactIdentity` 会处理什么

当前会处理：

- 电脑名；
- Windows 用户名；
- 已知用户目录路径，例如 `C:\Users\...`；
- 临时目录路径；
- 启动项里的用户字段；
- Chrome 和 Edge 扩展 ID；
- 注册表策略值里的本地身份路径；
- 控制台输出里的报告路径。

## 分享前检查

打开 `.txt` 报告，至少确认没有这些内容：

- 真实姓名或 Windows 用户名；
- 电脑名；
- `C:\Users\真实用户名\...` 这类路径；
- 浏览器扩展 ID；
- 账号、密码、验证码、支付信息；
- 私人文件名、聊天内容、邮件内容或浏览器历史。

如果你不确定某一行能不能公开，先删掉那一行，或者只分享相关片段。

## 可以保留的诊断信息

通常可以保留：

- Windows 版本；
- 设备型号；
- CPU、内存、磁盘使用情况；
- C 盘剩余空间比例；
- 启动项数量；
- 杀毒软件数量和状态；
- Windows Update 状态；
- 浏览器扩展数量；
- pending reboot 状态；
- `Quick Findings` 里的非身份化结论。

这些信息能帮助维护者判断问题方向。

## 在 GitHub issue 里怎么发

如果你要给本项目提交问题，请优先提供：

- 电脑症状：例如开机慢、浏览器卡、磁盘满、更新失败；
- Windows 版本；
- 是否使用了 `-RedactIdentity`；
- `.txt` 报告里和问题相关的片段；
- 你已经尝试过的步骤。

不要直接上传未检查的完整报告。不要上传包含私人信息的截图。

## 维护者边界

维护者可以帮你读报告和判断方向，但这个工具不会自动修复电脑。任何删除文件、禁用启动项、卸载软件、重置浏览器或改变安全设置的操作，都应该由电脑主人明确确认后再做。
